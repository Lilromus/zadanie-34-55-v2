<p class="ex">
<font face = "Arial" size = "5">1. Na co pozwala styl lokalny, który element będzie formatowany z użyciem stylu
lokalnego</font></br>
<font face = "Arial" size = "4">Styl lokalny pozwala na nadanie formatowania konkretnemu pojedynczemu elementowi</br>
strony. Jest wstawiany w tej samej linii, w której znajduje się element formatowany.</br>
O tym, który to będzie element, decyduje słowo kluczowe "selektor".</br>
</font>
</p>
<p class="ex">
<font face = "Arial" size = "5">2. Składnia teoretyczna</font></br>
<textarea>
&lt;selektor style="cecha1: wartość1; cecha2: wartość2..."&gt;...&lt;/selektor&gt;
</textarea></br>
<font size = "3" face = "Arial">lub inaczej </br></font>
<textarea>
&lt;selektor style="nasze style"&gt;treść bloku&lt;/selektor&gt;
</textarea>
</p>
<p class="ex">
<font face = "Arial" size = "5">3. Co może być selektorem</font></br>
<font size = "4" face = "Arial">Selektorem może być praktycznie dowolny znacznik, np. p (akapit), h1 (nagłówek), td</br>
(komórka tabeli) i inne. To właśnie elementom znajdującym się pomiędzy tymi znacznikami,</br>
nadajemy atrybuty formatowania.</font></br>
</p>
<p class="ex">
<font face = "Arial" size = "5">4. Co wipisujemy jako "cechę". Jakie jest znaczenie średników</font></br>
<font size = "4" face = "Arial">Jako "cecha" czy inaczej własność bądź właściwość (ang. "property") należy wpisać o jakie</br>
konkretnie atrybuty formatowania nam chodzi (np. kolor tekstu - color).</br>
Jednemu selektorowi możemy nadać kilka atrybutów (cech). Są one wtedy rozdzielone średnikami.</font></br>
</p>
<p class="ex">
<font face = "Arial" size = "5"> 5. Co wipisujemy jako "wartość"</font></br>
<font size = "4" face = "Arial">Jako wyraz "wartość" (w deklaracji stylu) wpisujemy dokładną wartość atrybutu (np. dla</br>
koloru tekstu będzie to: red, blue czy też #31F4A5 itd.). Wartość zależy ściśle od cechy, po</br>
której stoi w deklaracji. </font></br>
</p>
